= HeteroLink

[License: Apache 2.0]
[ESP-IDF: v5.3+]
[Platform: ESP32 | ESP32-S3 | ESP32-C6]

*基于 ESP32 的通用型高速调试协处理器平台*

HeteroLink 为单板计算机、微控制器和嵌入式系统提供强大的调试、监控和数据采集能力。通过高速 SPI+DMA 和云端 MQTT 连接，实现 kHz 级遥测、μs 级响应和零代码即插即用的调试体验。

---

== 快速开发

=== 1. 环境准备

[source,bash]
----
# 安装 ESP-IDF (v5.3+)
git clone -b v5.3 --recursive https://github.com/espressif/esp-idf.git
cd esp-idf
./install.sh esp32 esp32s3 esp32c6
. export.sh

# 或使用 EIM (推荐)
cd ~/Code/HeteroLink/scripts
./install-esp-idf.sh
----

=== 2. 编译固件

[source,bash]
----
cd unit

# 设置目标芯片
idf.py set-target esp32c6

# 配置 WiFi 和 MQTT
idf.py menuconfig

# 编译
idf.py build
----

=== 3. 烧录测试

[source,bash]
----
# 烧录到设备
idf.py -p /dev/ttyUSB0 flash

# 监视串口输出
idf.py -p /dev/ttyUSB0 monitor
----

=== 4. 启动上位机

[source,bash]
----
cd software/host

# 编译并运行
cargo run --release

# 模拟模式（无需硬件）
cargo run --release -- --simulate
----

== 应用场景

* *电机参数在线整定* - 实时采集电流、电压、转速
* *实时示波器* - kHz 级采样，可视化波形
* *自动测试流程* - 脚本化测试序列
* *多设备集群管理* - 同时监控数十个节点
* *故障告警推送* - 异常即时通知

== MQTT 通信协议

=== Topic 规范

[cols="5*",options="header"]
|===
|Topic |方向 |QoS |Retain |说明

|`heterolink/subboard/{id}/status`
|设备 -> 云
|1
|是
|设备状态（online/offline + LWT 遗嘱）

|`heterolink/subboard/{id}/telemetry`
|设备 -> 云
|0
|否
|遥测数据

|`heterolink/subboard/{id}/command`
|云 -> 设备
|1
|否
|控制命令

|`heterolink/subboard/{id}/alarm`
|设备 -> 云
|1
|否
|告警通知

|`heterolink/subboard/{id}/response`
|设备 -> 云
|0
|否
|命令响应
|===

=== 消息格式

*状态消息* (`/status`):
[source,json]
----
{
  "state": "online",
  "timestamp": 1711094400,
  "firmware_version": "0.1.0",
  "uptime": 12345
}
----

*遥测数据* (`/telemetry`):
[source,json]
----
{
  "ts": 1711094400,
  "device_id": "subboard_001",
  "channels": {
    "ch1": 3300,
    "ch2": 2500,
    "ch3": 1800
  },
  "sample_rate": 1
}
----

*告警通知* (`/alarm`):
[source,json]
----
{
  "level": "critical",
  "channel": 1,
  "value": 3600,
  "threshold": 3500,
  "message": "Channel 1 exceeded threshold"
}
----

*控制命令* (`/command`):
[source,json]
----
{"cmd": "start", "params": {"sample_rate": 1000}}
{"cmd": "stop"}
{"cmd": "set_gpio", "channel": 4, "value": 1}
{"cmd": "set_alarm", "upper": 4000, "lower": 0, "enabled": true}
----

*命令响应* (`/response`):
[source,json]
----
{"cmd": "start", "status": "ok", "message": "Sampling started at 1000 Hz", "timestamp": 1711094400}
----
