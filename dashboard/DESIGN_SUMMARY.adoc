=== 3. 核心功能设计

==== 通信协议层
* *UART 通道*: 串口通信、二进制协议编解码、CRC16 校验
* *MQTT 通道*: MQTT 5.0 连接、Topic 管理、QoS 支持
* *协议定义*: 帧结构、命令字、遥测数据格式

==== 业务逻辑层
* *设备管理*: 多设备管理、连接状态、自动发现
* *数据处理*: 环形缓冲区、统计分析、数据导出
* *告警系统*: 阈值监控、告警触发、历史记录

==== 用户界面
* *主窗口*: 菜单栏、工具栏、状态栏、三分割布局
* *设备面板*: 设备列表、串口选择、连接控制
* *数据可视化*: 波形图 (QCustomPlot)、数据表格
* *配置面板*: UART 配置、MQTT 配置、告警阈值

==== 数据存储
* *配置管理*: JSON 格式、嵌套配置、默认值
* *数据记录*: CSV 导出、自动分段、文件大小限制

== 📐 协议设计要点

=== UART 二进制协议
* 帧头：`0xAA55` (小端)
* 最小帧长：8 bytes
* 最大载荷：65535 bytes
* 校验：CRC16/MODBUS
* 波特率：921600 (默认)

=== MQTT Topic 规范
....
heterolink/subboard/{device_id}/
├── status       # 设备状态
├── telemetry    # 遥测数据
├── command      # 控制命令 (订阅)
├── alarm        # 告警通知
└── config       # 配置 (订阅)
....

== 📝 备注

. *图标资源*: 需要从 Material Design Icons 或类似库获取
. *MQTT 库*: Qt 官方 QMqtt 需要 Qt 商业版，开源版可用 Paho MQTT C++
. *图表库*: QCustomPlot 是 GPL 许可，商业项目需考虑替代方案 (如 Qt Charts)
. *跨平台打包*: Windows 使用 windeployqt, Linux 使用 linuxdeployqt, macOS 使用 macdeployqt
