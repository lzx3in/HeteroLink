= HeteroLink 固件配置指南

通过 menuconfig 和 sdkconfig 文件配置固件。

== 🛠️ 配置方法

=== 方法 1: menuconfig（推荐）

[source,bash]
----
cd firmware/esp32/subboard
idf.py menuconfig
----

**导航**:
- 使用方向键浏览菜单
- 按 Enter 进入子菜单
- 按 Y/N 启用/禁用选项
- 按 Esc 返回上级菜单
- 按 S 保存配置
- 按 Q 退出

=== 方法 2: 编辑 sdkconfig.defaults

[source,bash]
----
# 编辑默认配置
vim sdkconfig.defaults

# 重新生成 sdkconfig
idf.py reconfigure
----

**优点**: 版本控制友好，适合团队协作

=== 方法 3: 直接编辑 sdkconfig

⚠️ **不推荐**: `sdkconfig` 是自动生成的，手动编辑可能在重新配置时被覆盖。

**仅用于**: 临时测试，不提交到版本控制

== 📋 配置选项详解

=== WiFi Configuration

[cols="2,1,3",options="header"]
|===
|*选项* |*类型* |*说明*

|`HETERO_WIFI_SSID`
|string
|WiFi 网络名称（SSID）

|`HETERO_WIFI_PASSWORD`
|string
|WiFi 密码

|`HETERO_WIFI_AUTO_RECONNECT`
|bool
|断开后自动重连（默认：y）
|===

**示例**:
[source,ini]
----
CONFIG_HETERO_WIFI_SSID="MyHomeWiFi"
CONFIG_HETERO_WIFI_PASSWORD="secret123"
CONFIG_HETERO_WIFI_AUTO_RECONNECT=y
----

=== MQTT Configuration

[cols="2,1,3",options="header"]
|===
|*选项* |*类型* |*说明*

|`HETERO_MQTT_BROKER_URI`
|string
|MQTT Broker 地址

|`HETERO_MQTT_BROKER_PORT`
|int
|MQTT Broker 端口（默认：1883）

|`HETERO_MQTT_CLIENT_ID`
|string
|MQTT 客户端 ID（必须唯一）

|`HETERO_MQTT_USERNAME`
|string
|MQTT 用户名（可选）

|`HETERO_MQTT_PASSWORD`
|string
|MQTT 密码（可选）
|===

**公共 Broker 推荐**:

* EMQX Public: `mqtt://broker.emqx.io:1883`
* Mosquitto Test: `mqtt://test.mosquitto.org:1883`
* HiveMQ Public: `mqtt://broker.hivemq.com:1883`

**示例**:
[source,ini]
----
CONFIG_HETERO_MQTT_BROKER_URI="mqtt://broker.emqx.io"
CONFIG_HETERO_MQTT_BROKER_PORT=1883
CONFIG_HETERO_MQTT_CLIENT_ID="heterolink_subboard_001"
# CONFIG_HETERO_MQTT_USERNAME is not set
# CONFIG_HETERO_MQTT_PASSWORD is not set
----

=== Device Configuration

[cols="2,1,3",options="header"]
|===
|*选项* |*类型* |*说明*

|`HETERO_DEVICE_ID`
|string
|设备唯一标识符

|`HETERO_DATA_UPLOAD_INTERVAL`
|int
|数据上报间隔（毫秒，默认：1000）

|`HETERO_ENABLE_UART_PROTOCOL`
|bool
|启用 UART 二进制协议

|`HETERO_ENABLE_SPI_DMA`
|bool
|启用 SPI+DMA 高速传输

|`HETERO_ENABLE_ADC_GPIO_PROBE`
|bool
|启用 ADC/GPIO 探测功能
|===

**示例**:
[source,ini]
----
CONFIG_HETERO_DEVICE_ID="subboard_001"
CONFIG_HETERO_DATA_UPLOAD_INTERVAL=1000
CONFIG_HETERO_ENABLE_UART_PROTOCOL=y
CONFIG_HETERO_ENABLE_SPI_DMA=n
CONFIG_HETERO_ENABLE_ADC_GPIO_PROBE=y
----

=== Debug Configuration

[cols="2,1,3",options="header"]
|===
|*选项* |*类型* |*说明*

|`LOG_DEFAULT_LEVEL`
|int
|日志级别（0-4）

|`HETERO_LOG_LEVEL`
|int
|HeteroLink 组件日志级别
|===

**日志级别**:

* 0: `LOG_LEVEL_NONE` - 无日志
* 1: `LOG_LEVEL_ERROR` - 仅错误
* 2: `LOG_LEVEL_WARN` - 错误 + 警告
* 3: `LOG_LEVEL_INFO` - 错误 + 警告 + 信息（默认）
* 4: `LOG_LEVEL_DEBUG` - 所有日志

**示例**:
[source,ini]
----
CONFIG_LOG_DEFAULT_LEVEL_INFO=y
# CONFIG_LOG_DEFAULT_LEVEL_DEBUG is not set
CONFIG_HETERO_LOG_LEVEL=3
----

== 🔧 高级配置

=== 自定义 GPIO 引脚

编辑 `components/adc_gpio_probe/include/adc_gpio_probe.h`:

[source,c]
----
#define ADC_GPIO_PROBE_DEFAULT_CONFIG() { \
    .channels = { \
        { .channel_num = 0, .mode = PROBE_MODE_ADC, \
          .adc_channel = ADC1_CHANNEL_0,  // GPIO1
          .attenuation = ADC_ATTEN_DB_11, \
          .bit_width = ADC_WIDTH_BIT_12 }, \
        { .channel_num = 1, .mode = PROBE_MODE_GPIO_INPUT, \
          .gpio_num = GPIO_NUM_2,  // 自定义 GPIO
          .pull_up_en = false }, \
    }, \
    .channel_count = 2, \
    .sample_interval_ms = 10, \
}
----

=== 自定义 SPI 引脚

编辑 `components/spi_dma/include/spi_dma.h`:

[source,c]
----
#define SPI_DMA_DEFAULT_CONFIG() { \
    .host_id = SPI2_HOST, \
    .sclk_io_num = GPIO_NUM_7,   // 自定义 SCLK
    .miso_io_num = GPIO_NUM_8,   // 自定义 MISO
    .mosi_io_num = GPIO_NUM_9,   // 自定义 MOSI
    .cs_io_num = GPIO_NUM_10,    // 自定义 CS
    .dma_channel = 2, \
    .max_transfer_sz = 4096, \
    .clock_speed_hz = 10 * 1000 * 1000, \
}
----

=== 自定义 UART 配置

编辑 `components/uart_protocol/include/uart_protocol.h`:

[source,c]
----
#define UART_PROTOCOL_DEFAULT_CONFIG() { \
    .uart_num = UART_NUM_1, \
    .tx_io_num = GPIO_NUM_17,  // 自定义 TX
    .rx_io_num = GPIO_NUM_18,  // 自定义 RX
    .rts_io_num = -1, \
    .cts_io_num = -1, \
    .baud_rate = 921600,       // 自定义波特率
    .rx_buffer_size = 2048, \
    .tx_buffer_size = 1024, \
}
----

=== 更改 MQTT Topic

编辑 `main/app_main.c`:

[source,c]
----
// 修改 Topic 前缀
#define MQTT_TOPIC_PREFIX "heterolink/subboard/"

// 在代码中替换
snprintf(topic, sizeof(topic), "%s%s/status", 
         MQTT_TOPIC_PREFIX, CONFIG_HETERO_DEVICE_ID);
----

== ⚠️ 注意事项

=== 必须遵守

1. **Client ID 唯一性**: 每个设备必须有唯一的 `HETERO_MQTT_CLIENT_ID`
   - 重复的 Client ID 会导致设备互相踢下线
   - 建议使用：`{项目}_{设备类型}_{编号}` 格式
   - 示例：`heterolink_subboard_001`

2. **WiFi 频段**: ESP32 仅支持 2.4GHz WiFi
   - 5GHz WiFi 无法连接
   - 确保路由器开启了 2.4GHz 频段

3. **内存限制**: 启用所有功能会增加内存占用
   - 典型空闲堆：150-280KB
   - 如果内存不足，禁用未使用的功能

4. **功耗考虑**: 更短的上报间隔会增加功耗
   - 电池供电：建议 ≥5000ms
   - USB 供电：可设置为 100-1000ms

=== 最佳实践

- **开发阶段**: 使用 DEBUG 日志级别，方便排查问题
- **生产部署**: 使用 WARN 或 INFO 级别，减少日志开销
- **团队协作**: 使用 `sdkconfig.defaults` 管理配置，提交到版本控制
- **多设备**: 为每个设备创建独立的配置文件

=== 常见问题

**Q: 配置修改后不生效？**

A: 运行 `idf.py reconfigure` 重新生成 sdkconfig

**Q: 如何恢复默认配置？**

A: 删除 `sdkconfig` 文件，运行 `idf.py menuconfig` 重新配置

**Q: 如何查看当前配置？**

A: 运行 `idf.py read-project-config` 或查看 `sdkconfig` 文件

**Q: 如何备份配置？**

A: 运行 `idf.py save-defconfig` 保存为 defconfig 文件
