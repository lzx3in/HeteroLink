= HeteroLink Subboard - 快速开始指南

== 1. 环境准备

=== 安装 ESP-IDF

[source,bash]
----
# 克隆 ESP-IDF 仓库（v5.3 或更高版本）
git clone -b v5.3 https://github.com/espressif/esp-idf.git
cd esp-idf

# 安装工具链
./install.sh esp32s3  # 根据芯片型号选择：esp32, esp32s3, esp32c3, esp32c6

# 加载环境变量
. ./export.sh
----

=== 验证安装

[source,bash]
----
# 验证 IDF 版本
idf.py --version
# 预期：ESP-IDF v5.3.x

# 检查工具链
xtensa-esp32s3-elf-gcc --version
----

=== 系统要求

- **操作系统**: Linux (Ubuntu 20.04+), macOS, Windows (WSL2)
- **磁盘空间**: 约 10GB
- **Python**: 3.8 或更高版本
- **Git**: 最新版本

== 2. 配置项目

=== 设置目标芯片

[source,bash]
----
# 进入项目目录
cd ~/Code/HeteroLink/firmware/esp32/subboard

# 设置目标芯片（根据你的硬件选择）
idf.py set-target esp32s3
# 或：idf.py set-target esp32
# 或：idf.py set-target esp32c3
# 或：idf.py set-target esp32c6
----

⚠️ **注意**: `set-target` 会清除之前的编译产物，需要重新编译。

=== 配置 WiFi 和 MQTT

**方法 1: 使用 menuconfig（推荐）**

[source,bash]
----
idf.py menuconfig
----

在菜单中配置：
----
HeteroLink Configuration  --->
    WiFi SSID: <你的 WiFi 名称>
    WiFi Password: <你的 WiFi 密码>
    MQTT Broker URI: mqtt://broker.emqx.io
----

**方法 2: 编辑 sdkconfig.defaults**

[source,bash]
----
# 编辑配置文件
vim sdkconfig.defaults
----

添加以下内容：
[source,ini]
----
CONFIG_HETERO_WIFI_SSID="YourWiFi"
CONFIG_HETERO_WIFI_PASSWORD="YourPassword"
CONFIG_HETERO_MQTT_BROKER_URI="mqtt://broker.emqx.io"
CONFIG_HETERO_DEVICE_ID="subboard_001"
----

然后重新生成配置：
[source,bash]
----
idf.py reconfigure
----

=== MQTT Broker 推荐

[cols="1,1,2",options="header"]
|===
|*Broker* |*地址* |*说明*

|EMQX 公共
|`mqtt://broker.emqx.io:1883`
|稳定，适合测试

|Mosquitto 测试
|`mqtt://test.mosquitto.org:1883`
|公共测试服务器

|HiveMQ 公共
|`mqtt://broker.hivemq.com:1883`
|公共测试服务器

|本地 Broker
|`mqtt://192.168.1.100:1883`
|自建 MQTT 服务器
|===

== 3. 编译烧录

=== 编译

[source,bash]
----
# 编译项目（首次编译约 5-10 分钟）
idf.py build
----

**预期输出**:
```
Build complete!
```

**编译产物位置**: `build/` 目录

=== 烧录

[source,bash]
----
# 烧录到设备（替换为实际串口）
idf.py -p /dev/ttyUSB0 flash
----

**查找串口**:
[source,bash]
----
# Linux
ls /dev/ttyUSB*
# 或
ls /dev/ttyACM*

# macOS
ls /dev/cu.usbserial*

# Windows
# 使用设备管理器查看 COM 端口
----

=== 监视串口

[source,bash]
----
# 监视串口输出（115200 波特）
idf.py -p /dev/ttyUSB0 monitor
----

== 4. 测试 MQTT 通信

=== 订阅设备状态

打开新终端，订阅设备状态：
[source,bash]
----
# 订阅所有子板消息
mosquitto_sub -h broker.emqx.io -t "heterolink/subboard/#" -v

# 或只订阅特定设备
mosquitto_sub -h broker.emqx.io -t "heterolink/subboard/subboard_001/#" -v
----

**预期输出**:
```
heterolink/subboard/subboard_001/status {"state":"online","timestamp":1711094400,...}
heterolink/subboard/subboard_001/telemetry {"ts":1711094400,"channels":{"ch1":3.300,...}}
```

=== 发送控制命令

打开另一个终端，发送控制命令：
[source,bash]
----
# 启动采集
mosquitto_pub -h broker.emqx.io \
  -t "heterolink/subboard/subboard_001/command" \
  -m '{"cmd": "start"}'

# 停止采集
mosquitto_pub -h broker.emqx.io \
  -t "heterolink/subboard/subboard_001/command" \
  -m '{"cmd": "stop"}'
----

=== 使用 MQTTX 客户端（可选）

下载 https://mqttx.app/[MQTTX] 图形化客户端：
1. 新建连接：`broker.emqx.io:1883`
2. 订阅 Topic: `heterolink/subboard/#`
3. 发布命令到：`heterolink/subboard/subboard_001/command`

== 6. 常见问题

=== WiFi 连接失败

**症状**: 一直显示 "Connecting to WiFi..."

**解决方案**:
1. 检查 SSID/密码是否正确（区分大小写）
2. 确认 2.4GHz WiFi 可用（ESP32 不支持 5GHz）
3. 检查信号强度（靠近路由器）
4. 重启路由器
5. 查看日志中的错误码

=== MQTT 连接失败

**症状**: WiFi 已连接但 MQTT 无法连接

**解决方案**:
1. 确认 Broker 地址和端口正确
2. 检查防火墙规则（1883 端口是否开放）
3. 测试 Broker 可达性：`ping broker.emqx.io`
4. 检查 Client ID 是否唯一
5. 启用 DEBUG 日志查看详细错误

=== 串口无输出

**症状**: 烧录后串口没有任何输出

**解决方案**:
1. 检查串口线连接（TX/RX 是否接反）
2. 确认波特率正确（115200）
3. 检查串口权限：`sudo chmod 666 /dev/ttyUSB0`
4. 尝试更换 USB 线或串口

=== 编译失败

**症状**: 编译时报错

**解决方案**:
1. 确认 ESP-IDF 环境变量已加载：`. $HOME/esp/esp-idf/export.sh`
2. 清理编译产物：`idf.py fullclean`
3. 重新配置目标芯片：`idf.py set-target esp32s3`
4. 检查依赖：`idf.py reconfigure`
